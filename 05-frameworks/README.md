# 05 - Frameworks Oracle

## Objetivo

Aprender a diseñar frameworks reutilizables para procesamiento Batch y Near Real Time (NRT).

Un framework permite estandarizar procesos, reducir código duplicado y mejorar mantenibilidad.

---

# ¿Qué es un Framework?

Es una estructura reutilizable que define:

- Flujo
- Reglas
- Logging
- Manejo de errores
- Configuración
- Monitoreo

---

# Temas

## 01 - Batch Framework

Procesos programados.

Ejemplos:

- cierres
- conciliaciones
- migraciones

---

## 02 - Near Real Time Framework

Procesamiento continuo.

Ejemplos:

- COB_REC_NRT
- sincronizaciones
- eventos

---

## 03 - Logging Framework

Registro de ejecución.

Ejemplos:

- inicio
- fin
- métricas
- errores

---

## 04 - Error Handling Framework

Gestión centralizada de errores.

---

## 05 - Retry Pattern

Reintentos automáticos.

---

## 06 - Configuration Driven

Procesos controlados por tablas de configuración.

---

## 07 - Scheduler Framework

Automatización mediante DBMS_SCHEDULER.

---

## 08 - Monitoring

Control operativo.

Alertas.

Métricas.
